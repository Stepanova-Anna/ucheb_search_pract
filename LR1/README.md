## Примеры применения методов и методологий при решении ИТ-задач

### Методы

| № | Метод | ИТ-задача | Как применяется |
|---|-------|-----------|------------------|
| 1 | A/B-тестирование | Оптимизация конверсии на сайте | Двум группам пользователей показывают разные версии интерфейса; сравнивают метрики (CTR, продажи). |
| 2 | Код-ревью (обзор кода) | Обеспечение качества ПО | Разработчики проверяют код коллег до слияния в основную ветку: ищут ошибки, уязвимости, нарушения стиля. |
| 3 | Нагрузочное тестирование (JMeter) | Оценка отказоустойчивости сервера | Симулируют 10 000 одновременных запросов к API, измеряют время ответа и точку отказа. |
| 4 | Рекурсивный спуск (парсинг) | Разбор арифметических выражений | Написание парсера на Python, который обрабатывает вложенные скобки и операции согласно грамматике. |
| 5 | Метод главных компонент (PCA) | Сжатие признаков для ML-модели | Уменьшают размерность данных (например, 100 признаков → 10) без потери 95% дисперсии. |

### Методологии

| № | Методология | ИТ-задача | Как применяется |
|---|-------------|-----------|------------------|
| 1 | Agile (Scrum) | Разработка мобильного приложения с нечёткими требованиями | Спринты по 2 недели, бэклог, ежедневные стендапы, демо для заказчика после каждого спринта. |
| 2 | DevOps | Ускорение релизов и повышение стабильности | Внедрение CI/CD, инфраструктура как код (Terraform), мониторинг (Prometheus + Grafana), культура совместной ответственности. |
| 3 | Waterfall (каскадная) | Разработка системы для госоргана с фиксированным ТЗ | Этапы: анализ → проектирование → кодирование → тестирование → внедрение (жёсткий порядок, без возвратов). |
| 4 | ITIL | Управление инцидентами в крупном дата-центре | Внедрение процессов: регистрация заявок (Service Desk), категоризация, эскалация, SLA, пост-мортем. |
| 5 | RUP (Rational Unified Process) | Крупный корпоративный ERP-проект | Использование 4 фаз (Начало, Уточнение, Конструирование, Передача) и артефактов: бизнес-модель, варианты использования, архитектура. |

## Этап 3. Практическое применение: разработка авторского мини проекта

**1. Постановка задачи**

Разработать систему учёта обучающихся (Student Database System) с консольным интерфейсом, которая позволяет:

- Хранить информацию о студентах (ФИО, группа, дата рождения, email)
- Хранить информацию о дисциплинах (название, преподаватель)
- Фиксировать оценки студентов по дисциплинам (оценки от 2 до 5)
- Выполнять CRUD-операции над всеми сущностями
- Рассчитывать средний балл успеваемости студента
- Использовать нормализованную структуру базы данных SQLite

***Требования к системе***

| Требование | Описание |
|------------|----------|
| **Надёжность** | Корректная обработка ошибок ввода и целостности данных |
| **Расширяемость** | Возможность добавления новых функций без переписывания ядра |
| **Простота** | Интуитивно понятный консольный интерфейс |

---

**2. Выбранная методология и обоснование**

**Выбрана методология:** `Agile` (итеративная разработка)

***Обоснование выбора***

| Критерий | Обоснование |
|----------|-------------|
| **Неопределённость требований** | В процессе разработки могут появиться новые функции (например, поиск по группе, фильтрация оценок). Agile позволяет гибко добавлять их. |
| **Поэтапная демонстрация** | Каждый спринт даёт работающий инкремент продукта: сначала работа со студентами, затем с дисциплинами, затем с оценками. |
| **Обратная связь** | Заказчик (или преподаватель) может проверить систему после каждого этапа и предложить улучшения. |
| **Низкий риск** | Ошибки выявляются рано, так как тестирование происходит после каждого спринта. |
| **Простота масштабирования** | Agile-подход позволяет легко добавлять новые возможности (экспорт в CSV, графики успеваемости) в следующих итерациях. |

# Student Database System

Система учёта обучающихся с консольным интерфейсом и SQLite БД.

## Выбранная методология

**Agile (итеративная разработка)** — выбрана из-за гибкости, возможности поэтапной демонстрации и раннего получения обратной связи.

## Этапы разработки

1. **Спринт 1:** Проектирование и создание нормализованной БД (3 таблицы)
2. **Спринт 2:** CRUD операции для студентов и дисциплин
3. **Спринт 3:** Работа с оценками, JOIN, агрегация (средний балл)
4. **Спринт 4:** Консольный интерфейс и интеграция всех модулей

## Применённые методы

- Нормализация БД (3 нормальная форма)
- Параметризованные SQL-запросы (защита от SQL-инъекций)
- FOREIGN KEY с ON DELETE CASCADE
- Агрегация (AVG) и JOIN для отчётов
- Инкапсуляция (класс StudentDB)

## Возникшие сложности и их решение

| Сложность | Решение |
|-----------|---------|
| Дублирование email при вставке | Добавлен UNIQUE-ограничение + обработка IntegrityError |
| Оценки вне диапазона 2–5 | Добавлен CHECK в схему БД + валидация в методе add_grade() |
| Удаление студента с оценками | Использован ON DELETE CASCADE в FOREIGN KEY |
| Нечитаемый вывод JOIN | Форматирование строк через f-строки |
